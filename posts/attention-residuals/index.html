Why Residual Connection Needs Attention

April 2026

Understanding the PreNorm Dilution Problem and how Attention Residuals fix it

Almost every modern Large Language Model (LLM) uses the exact same foundational architecture: the PreNorm Residual Connection. It became the industry standard because it provides more stable training for deep networks compared to the older PostNorm architecture.

But there is a hidden catch. As models scale past hundreds of layers, this exact mechanism creates a mathematical bottleneck that essentially dilutes the impact of early signals in deeper layers.

Let's break down exactly why this happens and how a new technique called Attention Residuals fixes it.

The Current Setup: The PreNorm Standard

In a standard PreNorm architecture, the equation to move data from one layer to the next looks like this:

$$X_{l+1} = X_l + F_l(\mathrm{Norm}(X_l))$$
  • $X$: The residual stream (the data flowing through the network).
  • $\mathrm{Norm}$: Layer Normalization.
  • $F_l$: The transformation function (e.g., Self-Attention or a Feed-Forward Network).

The key idea is simple: each layer adds its output to whatever came before.

If every modern LLM uses this, then what exactly is the problem?

The Problem: Signal Dilution at Depth

Let's unroll the recurrence and trace the data to see what is happening mathematically.

Assuming $X_0$ is our initial input:

$$X_{1} = X_0 + F_0(\mathrm{Norm}(X_0))$$
$$X_{2} = X_1 + F_1(\mathrm{Norm}(X_1))$$
$$X_{3} = X_2 + F_2(\mathrm{Norm}(X_2))$$

If we substitute backward and generalize all the way to a deep layer $L$, the residual stream is just a sum of every layer's contribution:

$$X_{l} = X_0 + \sum_{i=0}^{l-1} F_i(\mathrm{Norm}(X_i))$$

If we look closely at that equation, we find that every single previous layer contributes to the current state with an exact, fixed weight of $1$. There is no mechanism for the network to say "Layer 3's output matters more here than Layer 47's." They are all treated equally.

We can think of this residual stream as a uniformly distributed, weighted backpack. Every single layer throws its output into the backpack. Because we are endlessly adding these vectors together without scaling them, the mathematical magnitude (variance) of the hidden state grows linearly with the depth of the network. It scales as $\mathcal{O}(L)$.

Why This Breaks Things?

By the time the data reaches the deeper layers, that backpack is massive. But before those deeper layers can actually access and process this data, it must pass through Layer Normalization.

The residual stream is a sum of $L$ terms, each of roughly unit variance (due to the Norm before each $F_l$). The total variance is ~ $L$, so the standard deviation is ~ $\sqrt{L}$. When LayerNorm divides the stream by $\sqrt{L}$ to bring the variance back to 1, the magnitude of each individual layer's signal is divided by $\sqrt{L}$, meaning its proportion of the total variance shrinks to $\frac{1}{L}$.

The deeper the network, the more each early layer's signal is drowned out. By layer 200, the original token embeddings and the features learned in the first few layers are essentially noise - their signal magnitude has been reduced by a factor of roughly 14 (i.e., $\sqrt{200}$), reducing their variance share to just $\frac{1}{200}$.

This is the PreNorm Dilution Problem.

Furthermore, because all weights are fixed to 1, the network has no way to regulate gradient flow across its depth. This leads to disproportionately massive gradients piling up in the earlier layers, causing training inefficiencies. Hence deeper layers are essentially forced to output increasingly large numbers just to be "heard" over the noise of the accumulated stream.

The Solution: Attention Residuals

So, what is the solution?

What if, instead of giving every layer a fixed weight of 1, we let the network learn how much each previous layer should contribute?

Here is the current "addition" PreNorm formula:

$$X_{l} = X_0 + \sum_{i=0}^{l-1} F_i(\mathrm{Norm}(X_i))$$

Here is the Attention Residuals version:

$$X_{l} = \alpha_{0 \to l} \cdot X_0 + \sum_{i=1}^{l-1} \alpha_{i \to l} \cdot F_i(\mathrm{Norm}(X_i))$$

Instead of a fixed weight of $1$, we dynamically calculate an attention weight $\alpha_{i \to l}$ for every past layer $i$ contributing to the current layer $l$. Simple, right?

But how do we actually calculate these attention scores?

The scores are computed using a familiar dot-product attention formula:

$$\alpha_{i \to l} = \mathrm{Softmax}_i\left(\frac{Q_l \cdot \mathrm{RMSNorm}(K_i)}{\sqrt{d_k}}\right)$$

where the Softmax is taken over all previous layers $i = 0, 1, \ldots, l-1$.

Wait a second... what are the queries, keys, and values here? Let's go through this equation piece by piece.

1. The Pseudo-Query ($Q_l$)

Instead of generating a query dynamically from the current data flowing through the network, the authors assigned a single, permanent, learnable vector to each layer called a Pseudo-Query ($W_l$).

$$Q_l = W_l$$

We use this query to determine the similarity between what the current layer needs and what the previous layers (the keys) have produced.

Why does a fixed vector work here instead of a data-dependent query? Because the query isn't asking "what does this specific token need?" - it's asking a much simpler structural question: "Given this layer's role in the network, which previous layers' outputs are most useful?" That question depends on the layer's position and function in the architecture, not on the specific input data flowing through it. A fixed learned vector per layer is sufficient to capture this.

2. The RMSNorm on Keys ($K_i$)

The keys and values are nothing but the raw outputs of the previous layers.

However, if you recall we mentioned earlier that as we go deeper into the network, layers naturally tend to output mathematically larger numbers. If we use the raw outputs exactly as they are, the Softmax function will automatically assign higher weights to the deeper layers simply because their dot products with the query are larger, allowing them to unfairly dominate the attention weights.

To fix this, we apply Root Mean Square Normalization (RMSNorm) to the keys. This removes the impact of their magnitude and leaves only their feature direction.

3. The Softmax Competition

Finally, we compute the dot product between the Query and the Keys, and pass it through a Softmax function.

Softmax forces all the attention weights to sum up to exactly $1$. This creates a zero-sum competition among the layers. Because the weights must sum to $1$, the new hidden state remains a perfectly balanced weighted average. The magnitude of the residual stream no longer grows with depth - and that is how we solve the PreNorm Dilution Problem.

Problems with Full Attention Residuals

While the math is beautiful, it is practically very difficult to compute Full Attention Residuals for complex, deep networks.

Activation Recomputation: In standard models, once Layer 5 finishes passing its data to Layer 6, you can clear Layer 5's massive activation tensors from the GPU's immediate memory. But in Full Attention Residuals, Layer 100 needs to look directly at Layer 5. This means you must keep the entire history of every single layer's output alive in the GPU memory at all times. For a model with $L$ layers and hidden dimension $d$, this memory cost scales as $\mathcal{O}(Ld)$.

Pipeline Parallelism: Massive LLMs are too big for one GPU, so they are split across multiple GPUs in a chain. If GPU #2 needs to calculate its attention scores, GPU #1 must transmit the entire history of $L$ individual layer outputs across the network cables. The memory and communication overhead scales drastically as $\mathcal{O}(Ld)$.

The Fix: Block Attention Residuals

To solve these two problems, the authors introduced Block Attention Residuals.

Instead of doing attention over every single individual layer, they group the network into manageable chunks, or "blocks" (for example, blocks of $N=8$ layers).

Inside the block, the network turns off the expensive attention mechanism and reverts to the standard PreNorm addition. The layers compress themselves into a single "Block Summary."

Between the blocks, the network calculates the attention scores.

$$B_0 = X_0 \quad \text{(the initial embedding)}$$
$$B_1 = F_1(\mathrm{Norm}(X_1)) + F_2(\mathrm{Norm}(X_2)) + \cdots + F_8(\mathrm{Norm}(X_8))$$
$$B_2 = F_9(\mathrm{Norm}(X_9)) + F_{10}(\mathrm{Norm}(X_{10})) + \cdots + F_{16}(\mathrm{Norm}(X_{16}))$$

Then we compute the attention scores between these block summaries:

$$X_l = \sum_{j=0}^{B-1} \alpha_{j \to l} \cdot B_j$$

Yes, dilution still exists within each block - the 8 layers inside still use dumb addition. But the dilution from 8 layers is mild and manageable, a far cry from the catastrophic dilution across 200+ layers. The attention mechanism between blocks is what prevents that dilution from compounding across the full depth of the network.

Why RMSNorm is Crucial Here: Did you notice something? Because we use the "dumb addition" of previous layer outputs to create the block summary, deeper blocks will mathematically contain much larger numbers than earlier ones. This is exactly why the RMSNorm Shield is so vital. By applying RMSNorm to the Block Summaries before computing attention, it removes the magnitude discrepancy and keeps only the feature direction.

The memory and communication overhead reduces from $\mathcal{O}(Ld)$ to $\mathcal{O}(Bd)$, where $B$ is the number of blocks. For a 200-layer model with blocks of 8, that's a 25× reduction - the difference between a technique that's theoretically beautiful and one that can actually survive the constraints of real GPU clusters.

Closing Thoughts

The core insight of Attention Residuals is surprisingly simple: the residual stream's fixed, uniform weighting is a bottleneck, and replacing it with learned, normalized attention over layers fixes the dilution problem without sacrificing training stability. Block Attention Residuals make that insight practical at scale. It gives the model the retrieval capabilities of full attention across its depth, while actually fitting within the memory and communication budgets of real hardware.
